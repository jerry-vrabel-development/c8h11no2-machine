# v0.1 Spike — Audio-First Pull Prototype

> **Status:** Ready for development
> **Goal:** Prove the pull → reveal → reward audio loop feels incredible *before* building anything else.
> **Output:** A throwaway sandbox. Not production code. Will be rebuilt for v0.3.

---

## Why this spike exists

Audio is the one element of this project that cannot be designed on paper. The
reveal sequence is a 4-act composition (anticipation → drop → bloom → settle)
and every act needs to be heard, tuned, and re-heard before any other system is
built around it.

If the pull doesn't *feel incredible* after this spike, the entire project
needs rethinking. If it does, every subsequent decision rests on a known-good
foundation.

## What's in scope

A single page in a Vite + Tone.js sandbox containing:

1. **One button** labeled "PULL"
2. **A tier reveal display** showing the rolled tier (text + color, no card art yet)
3. **A molecule counter** that ticks up after each pull
4. **A "force tier" debug panel** for testing each tier reveal directly

That's it. No cards. No collection. No persistence. No badges. No PWA shell.

## What's explicitly out of scope

- Card art / sprites
- Creature names or families
- Collection storage
- Persistence (IndexedDB)
- PWA manifest / service worker
- Settings, ethics framing, first-run experience
- Visual polish beyond minimum needed to see what's happening
- Mobile optimization (desktop browser only is fine)
- Haptics (audio focus)

These come back in v0.3 once audio is locked.

## The 4-act reveal structure

Every pull plays out as four distinct sonic acts. They overlap slightly — the
next act begins as the previous one tails.

### Act 1 — Anticipation (0.0 – 0.3 to 1.5s, variable)

Plays while the system is "deciding." Variable length is the dopamine engine —
sometimes 300ms, sometimes 1.5s, weighted toward the lower end but with
occasional drawn-out builds.

- **Sonic vocabulary:** rising hum, coil charging, centrifuge spinning up
- **NOT:** musical phrases, casino bell ramps, slot machine reels
- **Frequency:** sweep from ~80Hz up to ~600Hz over the duration
- **Texture:** sawtooth or triangle wave, slight reverb, low-pass filter opening

### Act 2 — The Drop (single transient, ~50–100ms)

The moment of revelation. A single sound that punctuates the end of suspense.

- **Common/Uncommon:** soft, dampened
- **Rare+:** sharper, brighter
- **Mythic:** sub-bass impact with high-frequency sparkle
- All tiers share the *timing* of this transient — distinguishability happens in Act 3

### Act 3 — The Bloom (tier-specific, 0.4s – 4s)

This is where rarity is communicated. Each tier has a recognizable sonic
signature distinct from its neighbors.

| Tier | Length | Sonic character |
|---|---|---|
| Common | 0.4s | Single soft chime, decay tail, one note |
| Uncommon | 0.6s | Two-note chime, ascending interval, brighter |
| Rare | 1.0s | Sustained chord (major third), sparkle layer |
| Epic | 1.5s | Arpeggio + sub-bass pulse, slight wobble |
| Legendary | 2.5s | Short fanfare (synth-brass), sustained pad underneath |
| Mythic | 4.0s+ | Choral pad, rising arpeggio, sustained resolution |

Critical: a user with their eyes closed should be able to identify the tier
from Act 3 alone.

### Act 4 — The Settle (1.0s, every pull)

The reveal is over. Molecules tick into the counter. User reads the flavor.
Quiet, satisfied.

- **Molecule tick:** plays N times where N = molecules awarded
  - Precision counter click, Geiger-like, ~30ms each
  - Slight pitch variation between ticks (random ±2 semitones)
  - This sound plays 1000+ times per engaged user. **It must not fatigue.**
- **Final settle:** subtle low-frequency thump as counter completes

## Tier roll logic (for the spike)

Use a single weighted roll, no pity timer for now:

```
Common:    60.0%
Uncommon:  25.0%
Rare:      10.0%
Epic:       4.0%
Legendary:  0.9%
Mythic:     0.1%
```

Award molecules per tier roll (midpoint of the GDD range is fine):
- Common: 2, Uncommon: 6, Rare: 15, Epic: 35, Legendary: 100, Mythic: 350

## Debug controls (top-right of page)

A small panel for testing each tier directly without waiting for RNG:
- Buttons: "Force Common" | "Force Uncommon" | "Force Rare" | "Force Epic" | "Force Legendary" | "Force Mythic"
- Toggle: "Skip anticipation" (for rapid iteration on Acts 2–4)
- Slider: "Anticipation duration" (manual override for tuning)
- Reset button: zero out molecule counter

## Visual minimum (just enough to test audio)

- Dark background (`#0E1116`)
- Centered PULL button (large, satisfying)
- Tier name displayed in tier color when reveal completes
- Molecule counter top-center, monospaced
- Debug panel top-right, plain HTML
- No animations beyond the bare minimum to know what's happening

Visual polish comes in v0.3. This spike is about *ears*.

## Success criteria

Before declaring this spike done, all of the following must be true:

- [ ] All 6 tier reveals feel **distinct** with eyes closed
- [ ] Pulling 50 commons in a row does not become annoying
- [ ] Anticipation variable delay creates real suspense (test with a friend)
- [ ] Mythic reveal is the one you most want to share
- [ ] Molecule tick is *satisfying* after 100 ticks, not grating
- [ ] Common reveal is fast enough not to slow grinding (~1.5s total)
- [ ] Legendary reveal makes you stop and notice (~3.5s feels right)
- [ ] Audio respects browser autoplay rules (initialized on first user gesture)
- [ ] No clipping, no harsh transients above ~8kHz, no fatigue after 10 min

If any criterion fails, iterate on the relevant act before moving on.

## Technical stack (recommended, not mandatory)

- **Vite** vanilla TypeScript template (no React needed for this spike)
- **Tone.js** for all sound generation
- **No external sound files** — fully procedural so the synth parameters are
  the spec and easy to iterate
- **Single index.html + main.ts** — keep it embarrassingly simple
